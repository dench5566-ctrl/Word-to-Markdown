# d2 — Техническое задание

**DOCX → Markdown конвертер для Android / iOS**

**Статус:** Эталонный документ. Не подлежит произвольным изменениям без явного решения.

---

## 1. Общее описание

**d2** — минималистичное мобильное приложение для Android и iOS. Приложение позволяет выбрать файл `.docx` на устройстве, конвертирует его в чистый Markdown и сохраняет результат через системный диалог сохранения.

**Ключевые характеристики:**
- Один главный экран с четырьмя динамическими состояниями.
- Полностью оффлайн (не требует интернета).
- Все тексты интерфейса и визуальные параметры управляются через JSON-конфиги (`texts.json` и `appearance.json`).
- Apple-style премиум-минимализм: воздух, тонкие линии, спокойный интерфейс.
- Плавные анимированные переходы между состояниями.
- Главный сценарий максимально простой: одна большая кнопка → выбор файла → мгновенная конвертация → сохранение.

---

## 2. Главные принципы

- **Предельная простота.** Приложение делает ровно одну вещь и делает её хорошо. Минимум действий пользователя.
- **Production pipeline.** Проект создаётся как долгосрочная система: версионирование сборок (`d2_XXXX.zip`), полные архивы, лог изменений (`d2_log_ai.html`), воспроизводимость.
- **Конфигурируемость.** Ни один текст и ни один визуальный параметр не захардкоден в коде. Все данные загружаются из JSON-конфигов.
- **Clean Architecture.** Чёткое разделение ответственности:
  - `core/` — базовые инструменты и провайдеры;
  - `config/` — модели данных конфигов;
  - `features/` — вся бизнес-логика (сервисы);
  - `ui/` — только отображение.
- Логика невогда смешивается с UI.
- **Оффлайн-first.** Приложение полностью работоспособно без подключения к интернету.

## 5. Состояния приложения (AppState)

Один экран (`MainScreen`) с четырьмя состояниями. Переходы через `AnimatedSwitcher` + `FadeTransition` + `SizeTransition`.

### 5.1 idle (ожидание)
- Чистый белый/светлый фон.
- По центру — **одна большая кнопка** «Выбрать файл .docx».
- Под кнопкой — мелкий текст «Поддерживаются файлы до 15 МБ».
- Опционально: сворачиваемый блок «Последние конвертации».

### 5.2 converting (конвертация)
- Кнопка неактивна.
- Центральный `CircularProgressIndicator`.
- Текст «Конвертируем документ…».
- Всё остальное заблокировано.

### 5.3 result (успех)
- Зелёная иконка успеха (размер ~80 px).
- Заголовок «Файл успешно конвертирован».
- Имя исходного файла и имя результата (`.md`).
- Карточка с путем к файлу (`SelectableText`).
- Кнопки:
  - «Сохранить .md» (основная)
  - «Поделиться»
  - «Конвертировать другой файл»

### 5.4 error (ошибка)
- Сообщение об ошибке.
- Кнопки «Повторить» и «Назад».
- При возможности — показывать исоходный файл.

## 6. Логика конвертации

### 6.1 Алгоритм
1. Пользователь нажимает большую кнопку → открывается `file_picker` (только `.docx`).
2. Выбран файл → переход в состояние `converting`.
3. Чтение и конвертация файла через `DocxToMarkdownService`.
4. При ошибке → состояние `error` + сообщение.
5. При успехе → переход в `result`.
6. Пользователь нажимает «Сохранить .md» → `FileSaver.instance.saveAs()` с именем ` Оригинал.md`.
7. После успешного сохранения:
   - Инкремент счётчика (`CounterService`)
   - Запись в `conversions.log` (`LogService`)
   - Показать SnackBar «Файл сохранено».

### 6.2 DocxToMarkdownService
- Основной пакет: `docx_to_markdown`
- Поддерживаемые элементы (minimum):
  - Заголовки H1–H6
  - Жирный, курсив, подчёркивание
  - Маркированные и нумерованные списки
  - Абзацы и переносы строк
  - Простые таблицы (по возможности)
- Изображения: извлекать и сохранять отдельно (опционально в MVP) или игнорировать с комментарием.

### 6.3 Именование файлов
- Имя результата: `{ оригинальное_имя_без_расширения}.md`
- При совпадении имень — добавлять суффикс `_1`, `_2` и т.д. (или использовать счётчик как в m2).

### 6.4 Лог конвертаций
- Файл: `conversions.log` в директории документов приложения.
- Формат строки: `YYYY-MM-DD HH:MM — filename.md`
- Показ последних 5–10 записей в состояние `idle` (сворачиваемый блок).

## 9. Требования к качеству

### 9.1 Функциональные
- Конвертация работает полностью оффлайн.
- Маркдаун получается чистым и читаемом.
- Файл сохраняется с правильним именем.
- Кнопка «Поделиться» работает.
- Тёмная тема полностью поддерживается.

### 9.2 Нефункциональные
- Производительность: конвертация файла до 10–15 МБ должна занимать не более 3–5 секунд.
- Стабильность: приложение не падает на типичных .docx файлах.
- Респонсив: корректная работа на телефонах и планшетах.

### 9.3 Production pipeline
- Версионирование: `d2_0001.zip`, `d2_0002.zip`…
- Каждый архив содержит `d2_log_ai.html` (по аналогии с m2).
- Чистая архитектура и соответствие этому ТЗ.
